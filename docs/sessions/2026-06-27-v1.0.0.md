# Session 2026-06-27 — v1.0.0: in-prose formula hover + ⌘K search, a pre-release sweep, and the release cut

Resumed from the ADR-0026 parked items: in-prose formula-token hover ("the rendered-KaTeX-span → formula-id
mapping is ambiguous and would need a design pass") and a comprehensive search. The owner asked to build both,
do a final accuracy/leaked-rendering sweep, update the docs, and cut **v1.0.0**.

## Shipped — the formula-affordances layer (ADR-0030)

The design pass: **the span→formula mapping is authored and adjacent, not inferred.** An author tags an
inline-math span with the formula id written right after it — `$\tau = RC$@{em-rc-time-constant}`. No heuristics,
no false positives.

- **One dataset, two consumers.** `src/lib/search.js#buildSearchIndex()` aggregates the committed
  `formulas.json` + the solution JSONs (formula name/symbols/desc/domain + **build-baked inline KaTeX**, lesson
  title/topic/regime, pages) into a static `/search-index.json` endpoint, fetched once and cached (no KaTeX
  ships to the browser; no per-page blob).
- **`inline()` token (`src/lib/katex.js`).** A math span followed by `@{id}` renders byte-identically, then is
  wrapped in `<a class="ftok" data-fid="id" href="/reference/#id">` with the tag stripped.
- **The global island (`src/islands/SiteSearch.svelte`, `client:load` in `Base.astro`):** a **⌘K / Ctrl-K
  command palette** (substring match + light ranking, arrow-keys + Enter, KaTeX-previewed results; header trigger
  collapses to an icon on mobile) **and** the **in-prose hover popover** (document-level delegation on
  `[data-fid]`, robust against late-hydrated island prose).
- **An 8th build gate** `check-prose-links.mjs` (wired into `validate`): every `@{id}`/`data-fid` in committed
  solution JSON + authored Astro pages must resolve to a real formula. `validate-reference.mjs` gained a guard
  that every `formula.lessons` slug is a real lesson route.
- **Demonstration tags:** the guide's regime/one-idea prose, plus one canonical formula in five lessons spanning
  five domains (rc-charging, thin-lens, isobaric-work, standing-wave, decay).

Verified live (preview tools, `preview_eval`): palette opens on ⌘K, returns mixed formula/lesson results with
rendered KaTeX, arrow-keys + Enter navigate, Esc closes; a tagged in-prose token renders KaTeX with no `@{` leak
and shows the popover ("RC time constant") on hover; both light + dark; mobile trigger collapses; no console
errors.

## Shipped — a multi-agent pre-release sweep, and the fixes

Ran a background `Workflow` fan-out: per-domain physics/content accuracy (5 agents, adversarially verified),
leaked-rendering, link/nav integrity, doc-count consistency. 5 accuracy findings (all verified real, 0 rejected),
3 rendering, 2 link, 10 doc. Fixes:

- **coulomb-pe:** the charges were like-signed (both +2 μC) but the lesson narrated them as opposite/attracting
  with a "binding energy" / escape analogy. The model treats `q₁,q₂` as magnitudes, so the fix is textual: state
  "two opposite point charges, of magnitudes `q₁ = q₂ = 2 μC`" (answer unchanged) and reword the practice prompt.
- **faraday-generator:** "EMF a quarter cycle *ahead* of the flux" inverted the lead/lag reading (the EMF, a
  sine, *lags* the cosine flux). Reworded the scenario + misconception to "lags / a quarter cycle after."
- **diverging-lens:** the misconception placed the virtual image "behind the object" and conflated image with
  focus; corrected to "a virtual image that always sits between the lens and the object."
- **thin-lens:** the ray-step latex hard-coded `parallel↔F`, which matched the diverging prose/diagram but
  contradicted the converging case. Made the latex **lens-type-aware** in `thin_lens.py` (`parallel↔F'` for a
  converging lens).
- **Rendering:** the Astro word-glue gotcha on the guide foot (`its`/`the` + line-leading link) → `{" "}`;
  Python `**`/`*` unit notation leaking into the reference + result cells → a build-time `prettyUnit()`
  (`src/lib/view.js`) renders `m/s²`, `N·m²/C²`, `kg·m²`; authoring markup (`$…$`, the `@{id}` tags, `**`)
  leaking into lesson `<meta name="description">` → a `plain()`-projected description, and the raw `scenario` is
  now dropped from the player view so it never ships in island hydration props.
- **Links:** 10 formula `lessons` entries used the problem id instead of the route slug (`spring-damped`→`damped`,
  `freefall-drag`→`terminal-velocity`, `freefall-throw-up`→`throw-up`, `spring-shm`→`mass-spring`) — their
  reference "used in" links rendered as dead text. Fixed + guarded by the new validator.
- **Docs:** stale counts across ROADMAP/AGENTS/CHANGELOG reconciled to the v1.0.0 ground truth; AGENTS.md's
  Current-state + Where-next narrative trimmed (history moved to CHANGELOG/DECISIONS); ADR-0030 added.
